fix(injection): reliable content_scripts injection + IPTorrents host fix

Inject on supported sites via declarative content_scripts instead of relying
solely on programmatic injection from the MV3 service worker. Under load the
SW could miss/delay the tabs.onUpdated event, and a failure between
probeInjection (which sets __servarrInjecting) and the lock release left the
page permanently stuck with no content script and no retry, so the icon never
appeared until a reload. This was the root cause of the recurring CI flakiness
(tvdb, allocine, metacritic, ...) and an occasional missed icon for users.

- Declare the integration domains as content_scripts (run_at document_idle) in
  both the Chromium MV3 and Firefox MV2 manifests; the browser injects them
  reliably with no service-worker race.
- Background scripts: skip the programmatic path on domains already covered by
  content_scripts (avoids double injection); clear __servarrInjecting and allow
  a retry if an injection is interrupted. The programmatic path is retained for
  configured Servarr instance domains (unknown at install).
- index.js: reset the engine registry on (re)load so engines can't accumulate
  duplicates if the content script is ever injected twice.
- Fix the IPTorrents integration host match, which never matched any page (the
  pattern was missing its top-level domain). Now covers .com/.net/.me/.eu/.ru
  and the engine matches any IPTorrents TLD.
- Update REVIEWER_GUIDE (injection model + IPTorrents patterns) and CHANGELOG.

Validated locally: full Playwright suite 32 passed / 2 skipped / 0 flaky, and
the previously-flaky specs pass 30/30 under parallel load with retries off.
