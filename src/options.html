Advanced settings

These settings affect how this extension will identify the search page on the Servarr instance and how the search field on that page will be identified.

If an API key is entered on the settings page and the 'Auto populate from API' version is enabled then these values will be automatically populated based on the Servarr instance and version you are running. This is the recommended way to configure the extension. Auto population will only work if the API call works, otherwise default values will be used.

To set these settings manually turn off the 'Auto populate from API' switch and enter the desired values. If you find that this is necessary please raise an issue on GitHub with your Servarr instance and version number so that the extension can be updated for all users.

Search path URL

This should be the path (less protocol, domain and port) of the search page.

Search field selector

This is a jQuery selector that is used to find the search input field on the search page.

Custom icon settings

All integrations have a configured location where a servarr icon will be injected if a search term is found on the browsed page. These settings will override these locations and instead show an icon anchored to the side of the browser window or floating at a specified position.

Once enabled, an example icon should appear on this page which will change in real time based on the toggles and fields below.

If custom icons are enabled, they will only be shown if a single instance of the usual image container is found. In situations where multiple elements are found on the page, the integration's configured icon containers will be used as a fallback. Sometimes this will be obvious with multiple icons being injected on a page, but in other cases a fallback may be caused by a selector being found multiple times even though an icon is only injected once.
